계산 흐름 시각화

입력 → 공제 → 과세표준 → 세액 → 결과

각 단계별 색상을 따라 현재 위치와 다음 단계를 확인하세요.

입력 패널로 이동
Step 1 · 입력
입력 총급여·매출·이자/배당 금액을 빠른 입력 패널에 채워 주세요.
Step 2 · 공제
공제/가산 카드/연금/접대비/경비를 채워 공제 한도와 가산세를 반영합니다.
Step 3 · 과세표준
과세표준 원본 엔진에서 산출된 과세표준과 비교과세 결과를 읽어옵니다.
Step 4 · 세액
세액 계산 국세/지방세, Gross-up, 최저한세를 포함해 세액 흐름을 계산합니다.
Step 5 · 결과
환급/추납 환급/추납·비교과세 결과를 요약하고 공유 카드에 전달합니다.

배포 가이드

GitHub → Cloudflare Pages

정적 사이트라 빌드 없이 배포 가능합니다. 하위 폴더는 각각 원본 엔진을 포함합니다.

  1. 루트에 index.html, styles.css, app.js, 그리고 yearend/corporate/financial 폴더가 있는 상태로 GitHub에 올립니다.
  2. Cloudflare Pages: Build command 없음, Output directory /.
  3. 배포 후 탭별 임베드와 “새 창에서 열기” 버튼이 모두 작동하는지 확인합니다.

주민세 계산법

주민세는 산출세액의 10%를 기본으로 하며, 일부 감면·비과세 항목은 지자체 규정에 따릅니다.

  • 연말정산/법인세 결과의 국세 산출세액 × 10%를 기본 지방소득세로 반영합니다.
  • 과세표준 구간별 감면이 있는 경우 해당 지자체 공지와 납부서 기준을 확인하세요.
  • 지방교육세 등 부가세가 붙는 경우는 별도 계산 탭(원본 엔진)에서 확인할 수 있습니다.

WCAG 대비/접근성 안내

테마/대비 슬라이더로 색상 대비를 조정하고, 키보드와 스크린리더로 탐색할 수 있습니다.

  • 탭(Tab)과 시프트+탭으로 인터페이스를 이동하고, 엔터/스페이스로 버튼을 실행합니다.
  • 모든 상호작용 요소는 포커스 표시와 ARIA 라벨을 포함합니다.
  • 테마 토글과 대비 슬라이더를 사용해 밝기·채도를 조정해 WCAG AA 대비를 유지하세요.
  • 모달은 ESC 키와 배경 클릭으로 닫을 수 있으며, 열릴 때 포커스를 내부로 이동합니다.
