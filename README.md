색상 대비 강화와 사용자 정의 테마

"현재 파스텔 색상과 흰색 카드가 아름답지만, 모바일에서 작은 글자는 대비가 약할 수 있습니다. CSS 변수로 정의된 색상 팔레트를 이용해 사용자가 대비 강도를 조절할 수 있는 슬라이더를 추가하고, 어두운 모드에서도 오렌지·분홍 그라데이션이 자연스럽게 보이도록 색상을 조정해 주세요."

단계 네비게이션 강조 개선

"계산 흐름 단계(입력→공제→과세표준→세액→결과) 표시 영역의 현재 단계는 테두리만 붉게 표시되어 가시성이 낮습니다
tax3.pages.dev
. 활성 단계 카드의 배경을 주요 색상으로 채우고, 이전/다음 단계는 흐릿하게 표시하여 사용자가 진행 상태를 직관적으로 알 수 있도록 개선하세요."

요약 패널 고정 및 접기 기능

"실시간 요약 영역을 화면 우측에 스크롤 고정시키고, 사용자가 패널을 접거나 펼칠 수 있는 토글을 제공해 주세요. 작은 화면에서는 패널을 자동으로 접어 입력 영역을 넓게 사용하도록 반응형 처리를 해 주세요."

데이터 시각화 추가

"결과 요약에서 환급/납부액과 세목별 비중을 원형 차트 혹은 막대 차트로 시각화하는 기능을 추가해 주세요. 각 세목의 색상은 현재 사용 중인 아이콘 색상과 일치시키고, 마우스를 올리면 금액과 비율을 툴팁으로 보여 주세요."

2.2 기능 확장

단순 입력 모드와 고급 입력 모드 구분

"빠른 입력 패턴 외에도 ‘고급 입력’ 버튼을 제공하여 사용자가 더 많은 공제 항목(나이, 부양가족, 의료·교육비, 주택자금, 기부금 등)을 입력할 수 있는 확장 패널을 열 수 있게 해 주세요."

데이터 저장 및 불러오기

"사용자가 입력한 데이터를 브라우저 로컬스토리지에 저장하고, ‘저장된 계산 불러오기’ 기능을 추가해 이전에 계산한 값들을 다시 불러올 수 있게 해 주세요. 삭제 기능도 제공해 개인정보를 직접 관리할 수 있도록 합니다."

결과 내보내기 및 공유

"‘OG 카드 만들기’ 기능 외에 결과를 PDF 또는 이미지 파일로 다운로드할 수 있는 버튼을 추가해 주세요. 공유 버튼을 통해 카카오톡·메일 등으로 직접 전송할 수 있도록 Web Share API 연동을 고려합니다."

모바일 최적화 추가 개선

"모바일 환경에서 입력 카드들이 너무 길어지지 않도록 아코디언 형태로 열고 닫을 수 있게 하고, 숫자 입력 시 모바일 키보드가 숫자 키패드로 자동 전환되도록 <input type='number'>를 사용해 주세요."

접근성 설명 페이지

"‘WCAG 대비 강화’ 버튼을 클릭하면 색상 대비와 접근성을 설명하는 모달 페이지를 표시하고, 키보드 전용 탐색 방법 및 스크린리더 친화적 태그 사용 등을 안내해 주세요."

2.3 유지보수와 성능

코드 스플리팅과 Lazy Loading

"각 계산기 모듈(근로소득, 법인세, 금융소득 등)을 코드 스플리팅하여 필요할 때만 로드하도록 구현해 주세요. React Lazy + Suspense를 이용하면 초기 로딩 속도를 개선할 수 있습니다."

테스트 및 데이터 업데이트 자동화

"새로운 UI/UX에 맞춰 E2E 테스트를 작성하고, 세율·공제 데이터 파일을 주기적으로 갱신할 수 있도록 cron 스크립트나 GitHub Actions를 활용한 자동화 파이프라인을 구성해 주세요."

접속 로그 및 오류 추적

"사용자 행동을 분석할 수 있도록 익명화된 이벤트 로그(페이지 방문, 버튼 클릭 등)를 남기고, Sentry나 다른 오류 추적 도구를 연동해 프론트엔드 오류를 실시간으로 모니터링하세요."
